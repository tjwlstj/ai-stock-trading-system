#### 주요 변경사항

## 4. 프론트엔드 보안 취약점 분석

### js-yaml 취약점 (CVE-2025-64718)

#### 상세 정보
- **현재 버전**: 4.1.0
- **패치 버전**: 4.1.1
- **심각도**: Moderate (CVSS 5.3)
- **CWE**: CWE-1321 (Prototype Pollution)
- **발견일**: 2025-11-14
- **업데이트일**: 2025-11-17

#### 취약점 설명
- Prototype pollution 취약점 (`__proto__`)
- 신뢰할 수 없는 YAML 문서 파싱 시 공격자가 결과 객체의 프로토타입을 수정 가능

#### 해결 방법
1. js-yaml 4.1.1로 업데이트
2. 서버에서 `node --disable-proto=delete` 사용
3. Deno 사용 (기본적으로 pollution 보호 활성화)

#### 영향 분석
- **경로**: eslint > @eslint/eslintrc > js-yaml
- **직접 의존성**: 아니오 (간접 의존성)
- **현재 프로젝트 영향**: 낮음 (개발 도구 의존성)
- **권장 조치**: `pnpm audit --fix`로 자동 패치

### body-parser 취약점 (CVE-2025-13466)

#### 상세 정보
- **현재 버전**: 2.2.0
- **패치 버전**: 2.2.1
- **심각도**: Moderate
- **발견일**: 2025-11-25
- **업데이트일**: 2025-11-25

#### 취약점 설명
- URL 인코딩 사용 시 DoS 취약점
- 매우 많은 수의 매개변수를 포함한 페이로드로 CPU 및 메모리 사용량 증가
- 기본 100KB 요청 크기 제한 내에서 수천 개의 매개변수 전송 가능

#### 영향 분석
- **경로**: eslint > @modelcontextprotocol/sdk > express > body-parser
- **직접 의존성**: 아니오 (간접 의존성)
- **현재 프로젝트 영향**: 낮음 (개발 도구 의존성)
- **권장 조치**: `pnpm audit --fix`로 자동 패치
